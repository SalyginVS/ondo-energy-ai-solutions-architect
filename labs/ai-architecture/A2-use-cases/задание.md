Минимум 10, цель 12-13 use cases. Охват доменов обязателен: billing, operations, customer service, SCADA/OT, HR/internal, IT.
Структура каждого use case — 13 колонок согласно README:
1. Use Case Name
2. Domain
3. Problem Description (конкретная боль ONDO, не generic)
4. AI/ML Approach (RAG / classification / prediction / generation / agent)
5. Expected Value (High/Med/Low + 1 фраза обоснования)
6. Complexity (High/Med/Low + 1 фраза обоснования)
7. Priority (P1/P2/P3 из Value/Complexity matrix)
8. Threat Analysis (≥3 OWASP LLM:2025 категории с severity)
9. TCO Estimate (3-year ROM по шаблону)
10. ROI Projection (конкретные цифры, не "сэкономим деньги")
11. SAP/ERP Integration (конкретные модули и data flows)
12. Compliance (GDPR, NIS2, украинские требования)
13. Build vs Buy (рекомендация с обоснованием)
Что я проверю при Hard Mentor Review:
По каждому use case спрошу: откуда цифры в ROI? Если ответ «примерно» — пересчитаем. TCO должен быть buildable из допущений, не с потолка.
Threat Analysis: ≥3 OWASP категории на use case. Severity должна коррелировать с контекстом — нельзя ставить LLM01 LOW для user-facing chatbot.
SAP-интеграция: не «интегрировать с SAP», а конкретно — SAP MM (Materials Management), SAP PM (Plant Maintenance), SAP CS (Customer Service), RFC/BAPI или REST API, read-only или write.
ONDO Energy specificity: буду резать любой use case, который звучит как «взял с Medium.com».
